<task type="auto" tdd="true">
  <name>Task 1: Create metadata generation logic</name>
  <files>src/metadata.py, tests/test_metadata.py</files>
  <behavior>
    - Test 1: Given a progress.json dict, generates a JSON report keyed by page number (1-indexed) with category, reason, and telemetry.
    - Test 2: For failed pages, explicitly sets the category to "Failed" and includes the error message in the reason.
    - Test 3: Injects Page Labels into a PyMuPDF document using format "{page_num} - {category}" (or "{page_num} - Failed").
  </behavior>
  <action>
Create `src/metadata.py` containing `generate_report(progress_data)` and `inject_pdf_metadata(input_pdf_path, output_pdf_path, report_data)`.
- `generate_report` processes the `progress.json` status dictionary, mapping page numbers (1-indexed, derived from "page_1" keys) to their category, reason, and telemetry data (processing time, retries). (Per D-03, D-04, D-05, D-06)
- Handle failed pages explicitly, setting category to "Failed" and reason to the error. (Per D-07)
- `inject_pdf_metadata` opens the `input_pdf_path` with `fitz`, sets the page labels using `doc.set_page_labels(read_labels)` per PyMuPDF API, and saves to `output_pdf_path`. The label format is `{page_num} - {category}`. (Per D-01, D-02, D-09)
- Implement tests in `tests/test_metadata.py`. (Requires `pytest` and `PyMuPDF`).
  </action>
  <verify>
    <automated>pytest tests/test_metadata.py</automated>
  </verify>
  <done>Tests pass for report generation and PDF label injection.</done>
</task>
